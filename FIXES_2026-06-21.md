# 🔧 Correções de Produção - 2026-06-21

## 1. Backend - Validação de CORS_ORIGINS

**Problema:** 
```
pydantic_settings.sources.SettingsError: error parsing value for field "cors_origins" from source "EnvSettingsSource"
json.decoder.JSONDecodeError: Expecting value: line 1 column 1 (char 0)
```

**Causa:** Pydantic v2 tentava fazer `json.loads()` em string simples

**Solução:**
Adicionado `@field_validator` em `backend/app/config.py`:
```python
@field_validator("cors_origins", mode="before")
@classmethod
def validate_cors_origins(cls, v):
    """Ensure cors_origins is always treated as a string, never JSON"""
    if isinstance(v, str):
        return v.strip()
    if v is None or v == "":
        return "http://localhost:3000"
    return str(v).strip()
```

## 2. Frontend - API URL em Produção

**Problema:**
Frontend continuava apontando para `http://localhost:8000` em produção

**Causa:**
NEXT_PUBLIC_* variáveis são compiladas em BUILD TIME, não em runtime

**Solução:**
- Adicionado `ARG NEXT_PUBLIC_API_URL` no `frontend/Dockerfile`
- Workflow passa valor correto em build: `NEXT_PUBLIC_API_URL=https://mobidf.brocode.net.br/api`

**Arquivo:** `frontend/Dockerfile`
```dockerfile
ARG NEXT_PUBLIC_API_URL=http://localhost:8000
ENV NEXT_PUBLIC_API_URL=${NEXT_PUBLIC_API_URL}
```

**Arquivo:** `.github/workflows/build-and-deploy.yml`
```yaml
build-args: |
  ...
  NEXT_PUBLIC_API_URL=https://mobidf.brocode.net.br/api
```

## 3. Docker Compose - Arquivo .env em Produção

**Problema:**
```
env file /home/github-runner/actions-runner/_work/mobidf-ai/mobidf-ai/.env not found
```

**Causa:**
`docker-compose.yml` procurava `.env` que não existe em produção

**Solução:**
- Removido `env_file: [.env]` de `docker-compose.yml` (dev usa auto-load)
- Mantido em `docker-compose.prod.yml` apenas (carrega `.env.production`)
